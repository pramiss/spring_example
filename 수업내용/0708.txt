1. 쇼핑몰 - 쇼핑몰, 중고거래
주요 구현: 사용자 인증 및 권한 부여, 상품목록 및 검색, 장바구니, 주문처리, 결제 게이트웨이 통합
2. MEMO 게시판 - 블로그
주요 구현: 사용자 등록 및 로그인, 글 작성 및 편집, 댓글 기능, 태그 및 카테고리 분류
도전 요소: 방문한 글의 태그를 기반으로 빅데이터를 통한 새로운 글 추천
3. SNS 웹사이트 - 인스타그램
주요 구현: 프로필 관리, 친구 추가 및 팔로우 기능, 게시물 작성 및 피드, 실시간 채팅
4. 온라인 예약 시스템
5. 케어 관리 시스템 - 헬스케어
주요 구현: 환자 기록 관리, 예약 시스템, 의사와 환자간 메시지, 건강 데이터 분석
도전 요소: 민감한 데이터 보호
6. 공유 플랫폼 - 레시피 공유 사이트
특징: 블로그보다 더 전문적인 분야의 모임


[프로젝트 순서]
기획서 -> (일정표) -> DB 설계 -> URL 설계 -> 구현 -> 테스트
: 설계와 테스트에 시간을 오래 들일 수록 완성도가 높아짐

MEMO 게시판 - 예제
SNS(인스타그램) - 퀴즈

[1. 기획서]
Figma로.

[2. DB설계]
3정규화를 기준으로 함.
1. 한 칼럼에 값이 여러개가 들어가지 않도록 함(1NF)
2. 한 테이블의 개념에 대해 한 행은 한 테이블에서 유일해야 함 => 테이블 분리
(ex. user 테이블에서 "1 배진하 영화" "2 배진하 축구"는 user에 대해 개념적으로 유일하지않음)
3. 테이블에 어울리는 컬럼만 저장

DB설계는 기획서를 기반으로 함.
집에서도 DB명, table명 일치시키기.

[3. URL 설계]
1. View 화면 URL 정의 (Front-End) - 뷰 화면 관점
2. API 인터페이스 SPEC 정의 (Back-End) - 데이트 흐름 관점

<주소 설계: 옛날방식>
/post-list
/post-detail?id=13...

<주소 설계: 요즘방식(Restful API)> - Method가 중요
/posts GET
/posts/post/13 GET (13번 게시글 조회)
/posts/post/13 PATCH, PUT (13번 게시글 수정)
/posts/post/13 DELETE (13번 게시글 삭제)
/posts/post     POST (새로운 글 작성)
